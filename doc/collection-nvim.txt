INTRODUCTION                                         *collection-introduction*

Config-collection-nvim is a collection of configurations aiming to provide
an IDE-like feel to neovim.

==============================================================================
FEATURES                                                 *collection-features*

- Asynchronous program running.
- Code formating.
- Toggling Terminal.
- Project specific settings with .vim.json.

==============================================================================
PREREQUISITES                                       *collection-prerequisites*

- Neovim 5.0

OPTIONS                                                   *collection-options*

g:collection_run_program                            *g:collection_run_program*

	Name of the command to be used for running the program. It should 
	start with uppercase letter.

	This command allows adding arguments. If filetype has compile and
	execute options set up, arguments starting with - will be added when
	compiling, others when executing. Example:
>
	:R -lm < test.in
<
	See |collection-filetypes| for more on setting up compilers, 
	interpreters... for filetypes.

        default value: "R"

g:collection_toggle_errorlist                  *g:collection_toggle_errorlist*

	This variable sets up the key for toggling the 'errorlist' buffer
	created when running the program.

	Toggling it off will keep the content and it will not kill the job 
	if the program hasn't finished running yet.

        default value: "<S-E>"

g:collection_errorlist_type                      *g:collection_errorlist_type*

	Type of window to be used when toggling an errorlist. Set this value 
	to 1 to use normal bottom split instead of vertical split.

        default value: 0

g:collection_errorlist_size                      *g:collection_errorlist_size*

	Window size to be used for errorlist when toggling. 
	Size can be set between 1 and 200.

        default value: 50

g:collection_format_file                            *g:collection_format_file*

	This variable sets up the key to be used to format the whole file.
	Formating uses default indenting if no other options are provided.

	See |collection-filetypes| for more on adding formaters for a specific 
	filetype.

        default value: "<Space>f"

g:collection_format_ignore                        *g:collection_format_ignore*

	List of comma separated filetypes that won't be formated.

        default value: "text,netrw,markdown"

g:collection_format_save                            *g:collection_format_save*

	Changing value of this option to 0 will disable saving the file when
	formating. 
	To automatically format on save, you can add your format key to a
	BufWritePre event.

        default value: 1

g:collection_toggle_terminal                    *g:collection_toggle_terminal*

	This variable sets up key to be used for toggling the terminal.
	Toggling the terminal off does not kill the process, so the terminal
	keeps the same instance.

        default value: "<F4>"

g:collection_terminal_type                        *g:collection_terminal_type*

	Type of terminal to be used when toggling. Set this value to 1 to
	use normal bottom split terminal instead of vertical split.

        default value: 0

g:collection_terminal_size                        *g:collection_terminal_size*

	Window size to be used for terminal when toggling. 
	Size can be set between 1 and 200.

        default value: 50

CONFIGURING FILETYPES                                   *collection-filetypes*

This plugin allows setting up compilers, interpreters, executions and 
formaters for each filetype.

Options can be added either to init.vim or to .vim.json file, which
allows different configurations for different projects. See |collection-json|

1. Interpreter                              *collection-filetypes-interpreter*

	Examples:
>
	let g:collection_python_interpreter="python3 %:p"
	let g:collection_javascript_interpreter="node %:p"
<
	The value of the interpreter option should start with the name of
	the interpreter and continue with paths and additional settings.
	Paths should be provided unexpanded. See |expand| . 

	Adding '.extension' to the unexpanded path will add the extension 
	after expanding:
>
	pathtofile/file.ts --> %:p:r.js --> pathrofile/file.js
>

2. Compiler                                    *collection-filetypes-compiler*

	Examples:
>
	let g:collection_java_compiler="javac %:p"
	let g:collection_c_compiler="gcc %:p -o %:p:r -Wall"
	let g:collection_cs_compiler="csc %:p"
<
	The value of the compiler option should start with the name of
	the compiler and continue with paths and additional settings.
	Paths should be provided unexpanded. See |expand| .

	Adding '.extension' to the unexpanded path will add the extension 
	after expanding:
>
	pathtofile/file.ts --> %:p:r.js --> pathrofile/file.js

3. Execute                                     *collection-filetypes-execute*

	This setting should be used for filetypes that need compiling. 
	Program will be executed only if the compiling has been successful.

	Examples:
>
	let g:collection_java_execute="java %:p"
	let g:collection_c_execute="%:p:r"
	let g:collection_cs_execute="mono %:p:r.exe"
<
	The value of the compile option should start with the name of
	the compiler and continue with paths and additional settings.
	Paths should be provided unexpanded. See |expand| .

	Adding '.extension' to the unexpanded path will add the extension 
	after expanding:
>
	pathtofile/file.ts --> %:p:r.js --> pathrofile/file.js

3. Formater                                    *collection-filetypes-formater*

	Formater to be used for the filetype when formating the file. If
	no option is provided, default indenting will be used.

	Examples:
>
	let g:collection_python_formater="autopep8 %"
	let g:collection_typescript_formater="prettier --use tabs %"
<
	The value of the formater option should start with the name of
	the formater and continue with paths and additional settings.
	Normally adding '%' as a path will suffice.

Adding .vim.json file                                        *collection-json*
	
	Filetype configurations can be added to .vim.json file in the root
	of the project.

	Configurations in json file override default settings in init.vim.

	Example .vim.json:
>
	{
	    "typescript": {
		"compiler": "tsc --project tsconfig.json",
		"formater": "prettier %"
	    }
	}
<
	You can add configurations for multiple filetypes in the json.
